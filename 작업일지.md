## 2026-03-01 (Day 1) — 프로젝트 생성 + 전체 기능 구현

### 컨셉

YouTube 알고리즘에 이끌려 쇼츠/도파민 콘텐츠를 소비하는 대신, **내가 존경하고 삶에 도움되는 채널과 영상만** 볼 수 있는 큐레이션 뷰어. MyRoutine과 동일한 단일 파일 PWA 패턴으로 구현.

### 의도적 제외 (안 만드는 것)
- YouTube 검색 (알고리즘 영역)
- 추천/트렌딩
- 쇼츠 표시
- 댓글
- 자동 재생 (다음 랜덤 영상)

### 수행 작업

1. **프로젝트 골격 생성**
   - `index.html` 단일 파일 PWA (HTML+CSS+JS, ~2,270줄)
   - `sw.js` Service Worker (네트워크-퍼스트 HTML, 캐시-퍼스트 기타)
   - `manifest.json` PWA 매니페스트 (YouTube 레드 테마)
   - 캐시 버전: `mytube-v1`

2. **CSS 디자인 시스템**
   - YouTube 레드 테마 CSS 변수 (`--red: #FF0000`)
   - 다크모드 변수 오버라이드 (`[data-theme="dark"]`)
   - 카드, 그리드, 칩, 모달, FAB, 토글 등 컴포넌트 스타일

3. **3탭 구조 구현**
   - 📺 내 영상: YouTube URL 붙여넣기 → 썸네일 그리드
   - 📡 내 채널: 채널 URL 추가 → 최신 영상 목록 (API 키 필요)
   - ▶️ 플레이어: 깔끔한 임베드 플레이어 (추천/쇼츠/댓글 없음)
   - 하단 탭 바 + 스와이프 제스처 전환

4. **내 영상 탭 (API 키 없이 동작)**
   - `parseYouTubeVideoId()` — 6가지 YouTube URL 포맷 파싱 (watch, youtu.be, embed, shorts, v, live, bare ID)
   - oEmbed API로 제목/채널명 자동 조회 (`fetchVideoInfo()`)
   - 영상 그리드 렌더링 (썸네일 + 제목 + 채널명)
   - 시청완료 토글 (`toggleWatched()`), 삭제 (`deleteVideo()`)
   - 영상 카드 클릭 → 플레이어 탭 자동 전환

5. **플레이어 탭**
   - YouTube IFrame Player API 동적 로드 (`loadYouTubeAPI()`)
   - `rel=0, playsinline=1, modestbranding=1, autoplay=1`
   - 영상 끝나면 자동 시청완료 표시 (`onVideoEnded()`)
   - 플레이어 하단 영상 정보 + 시청완료 버튼
   - API 미로드 시 fallback: youtube-nocookie.com iframe

6. **내 채널 탭 (YouTube Data API v3)**
   - `parseYouTubeChannelInfo()` — 3가지 채널 URL 포맷 (/channel/UC, /@handle, /c/user, bare @handle)
   - `fetchChannelById()`, `fetchChannelByHandle()`, `fetchChannelByCustom()`
   - 채널 클릭 → 슬라이드-인 오버레이로 최신 20개 영상 목록 (`openChannelVideos()`)
   - `fetchPlaylistVideos()` — uploadsPlaylistId로 영상 목록 + duration 조회
   - Shorts 필터링: `parseDuration()` ISO 8601 파싱, 60초 이하 제외
   - 채널 영상에서 "내 영상에 저장" 기능 (`saveChannelVideoToMyVideos()`)
   - API 키 미설정 시 안내 배너 + 설정 이동

7. **카테고리 + 검색**
   - 카테고리 CRUD (`addCategory()`, `deleteCategory()`)
   - 16개 이모지 아이콘 선택 그리드
   - 수평 스크롤 칩으로 필터링 (`renderCategoryChips()`, `filterCategory()`)
   - 검색바 토글 + 실시간 필터링 (`onSearchVideos()`, `onSearchChannels()`)

8. **설정 모달**
   - 다크모드 토글 (`toggleDarkMode()`, localStorage `myTubeDark`)
   - 언어 선택 (ko/ja/en)
   - YouTube API 키 입력 + 발급 가이드 (`showApiGuide()` — 5단계 안내)
   - Shorts 필터링 토글
   - 계정 (Google 로그인/로그아웃)
   - 앱 정보 (v1.0, Bekking Studio, 피드백 이메일)

9. **i18n 다국어 (ko/ja/en)**
   - `TRANSLATIONS` 객체 약 90개 키 × 3언어
   - `t(key, params)`, `applyStaticTranslations()`
   - `data-i18n`, `data-i18n-placeholder`, `data-i18n-title` 속성
   - `detectLang()` navigator.language 자동 감지

10. **Firebase Auth + Firestore 동기화**
    - MyRoutine Firebase 프로젝트 (`myroutine-bekking`) 공유
    - Firestore 컬렉션: `mytube-users/{uid}` (MyRoutine `users/{uid}`와 분리)
    - Google 로그인 + popup-blocked 시 redirect fallback
    - 3초 디바운스 동기화 (`debouncedSync()`)
    - 충돌 해결: cloud-only → download, local-only → upload, both → upload (기존 sync 있으면)
    - API 키는 localStorage에만 저장 (Firestore 미동기화)
    - 동기화 상태 표시 (spinner/✓/⚠)

11. **PWA**
    - `manifest.json` (standalone, portrait, YouTube 레드 테마)
    - Android `beforeinstallprompt` 설치 배너
    - iOS Safari 설치 안내 배너
    - Service Worker 등록

12. **보안**
    - `escHtml()` — DOM API 기반 HTML 이스케이프
    - `escAttr()` — 인라인 이벤트 핸들러 속성값 이스케이프
    - `sanitizeColor()` — 6자리 hex만 허용
    - CSP 메타태그 (script-src, img-src, frame-src 등)
    - `photoURL` DOM API 렌더링 (XSS 방지)

13. **앱 아이콘 생성**
    - Canvas API로 512×512, 192×192 아이콘 생성
    - YouTube 스타일 재생 버튼 + "MyTube" 텍스트
    - 다크 배경 + 라운드 코너

14. **GitHub Pages 배포**
    - 리포: https://github.com/Daniel-Sung/MyTube
    - 배포: https://daniel-sung.github.io/MyTube/

## 2026-03-01 (Day 2) — RSS 전환 + Firebase 분리 + 풀스크린 재생

### 수행 작업

1. **영상 재생 시 자동 풀스크린 + 가로 모드**
   - Fullscreen API + Screen Orientation API
   - 풀스크린 진입 시 `screen.orientation.lock('landscape')` 자동 전환
   - 풀스크린 닫기 버튼 오버레이
   - `manifest.json` orientation `"portrait-primary"` → `"any"` 변경

2. **채널 기능 RSS 전환 — API 키 불필요**
   - YouTube Data API v3 의존성 완전 제거
   - CORS 프록시 (`corsproxy.io` 우선, `allorigins.win` 폴백) 추가
   - `resolveChannelId()`: YouTube 페이지 HTML에서 `externalId` 추출로 채널 ID 해석
   - `fetchChannelRSS()`: YouTube RSS XML 피드 파싱으로 채널 영상 목록 조회 (최신 15개)
   - 제거: API 키 입력, Shorts 필터, API 발급 가이드 UI
   - CSP `connect-src`에 CORS 프록시 도메인 추가

3. **Service Worker 수정**
   - 크로스 오리진 요청(CORS 프록시 등) 바이패스 — SW가 가로채서 index.html 반환하던 버그 수정
   - 캐시 버전 `mytube-v1` → `mytube-v2`

4. **Firebase 프로젝트 분리**
   - `myroutine-bekking` 공유 → `mytube-bekking` 독립 프로젝트로 전환
   - 새 프로젝트: Authentication (Google), Firestore Database (프로덕션 모드)
   - Firestore 보안 규칙: `mytube-users/{userId}` 본인만 읽기/쓰기
   - Google Cloud Console: OAuth 승인된 JavaScript 원본에 `daniel-sung.github.io` 추가

5. **Google 로그인 버그 수정**
   - **근본 원인**: CSP `script-src`에 `https://apis.google.com` 누락 → Google Auth 스크립트 차단 → `auth/internal-error`
   - CSP에 `apis.google.com` 추가로 해결
   - 로그인 방식: `signInWithPopup` 우선, `popup-blocked` 시 `signInWithRedirect` 폴백
   - `getRedirectResult()` 에러 캐치 + 토스트 표시 추가

### 버그 수정

1. **SW가 CORS 프록시 요청 가로채기**
   - 증상: `corsGet()` 호출 시 YouTube HTML 대신 index.html 반환
   - 원인: SW fetch 이벤트가 크로스 오리진 요청도 캐시-퍼스트로 처리
   - 수정: `url.origin !== self.location.origin` 이면 SW 바이패스

2. **채널 ID 해석 오류 (VEVO 채널로 잘못 해석)**
   - 증상: `@RickAstleyYT` → RickAstleyVEVO 채널로 해석
   - 원인: `"channelId"` 패턴이 페이지 내 다른 채널 ID를 먼저 매칭
   - 수정: `"externalId"` 패턴을 최우선 검사 (페이지 소유자의 실제 채널 ID)

3. **Google 로그인 `auth/internal-error`**
   - 증상: 모바일/데스크톱 모두 로그인 실패
   - 원인: CSP `script-src`에 `https://apis.google.com` 누락
   - 수정: CSP에 해당 도메인 추가

6. **MyRoutine CSP 수정 (연쇄 영향)**
   - MyTube와 동일한 CSP 문제 (`apis.google.com` 누락)가 MyRoutine에도 존재
   - 브라우저 CSP 적용 강화로 인해 동시에 발현된 것으로 추정
   - MyRoutine 레포 루트 `index.html` CSP에 `apis.google.com` 추가하여 해결

## 2026-03-04 (Day 3) — 오버레이 플레이어, URL 복사, 자막, 시청 통계, 채널 알고리즘 개선, 아이콘 변경

### 수행 작업

1. **PWA 아이콘 변경**
   - `MyTube.PNG` (디자인 원본) → `icon-192.png` (192×192), `icon-512.png` (512×512) 리사이즈
   - sharp-cli로 변환 (Canvas API 생성 → 실제 디자인 이미지로 교체)
   - `sw.js` 캐시 버전 `mytube-v2` → `mytube-v3`

2. **플레이어 탭 삭제 → 오버레이 플레이어**
   - `sec-player` HTML 섹션 삭제, Player 탭 버튼 삭제, `TAB_ORDER`에서 `'player'` 제거
   - `player-overlay` 추가 (channel-videos-overlay와 동일한 패턴의 전체화면 오버레이)
   - 뒤로가기 버튼, YouTube 플레이어, 영상 정보, 액션 버튼 포함
   - 풀스크린/가로모드 잠금 기존 로직 유지
   - `playVideo()`, `playVideoById()` → 탭 전환 대신 오버레이 열기
   - 새 함수: `openPlayerOverlay()`, `closePlayerOverlay()`
   - 오버레이 닫을 때 플레이어 정지 + 시청 시간 기록

3. **영상 URL 복사 기능**
   - 영상 카드, 채널 영상 카드, 플레이어 오버레이에 🔗 버튼 추가
   - `copyVideoUrl(videoId)`: `navigator.clipboard.writeText()` + `fallbackCopy()` (textarea execCommand)
   - i18n: `toast.urlCopied` 3개 언어 추가

4. **자막/스크립트 가져오기**
   - CORS 프록시로 영상 페이지 HTML 가져와서 `ytInitialPlayerResponse.captions.captionTracks` 추출
   - 자막 XML 가져와서 DOMParser로 파싱 → 타임스탬프 + 텍스트 오버레이 표시
   - 다국어 자막 있으면 `<select>` 언어 선택 UI
   - 타임스탬프 클릭 시 `seekPlayerTo()` → 해당 위치로 이동
   - 영상 카드, 채널 영상, 플레이어에 📝 버튼 추가
   - `transcript-overlay` 전체화면 오버레이 (transcript-header + transcript-body)
   - i18n: `transcript.*` 3개 언어 추가

5. **시청 시간 로그 (설정 내 통계)**
   - `data.watchLog = { totalSeconds, channels: {name: secs}, videos: {id: secs} }` 추가
   - YouTube Player API `onStateChange` 확장: PLAYING → 시작 시간 기록, PAUSED/ENDED → 경과 시간 누적
   - `accumulateWatchTime()`, `stopWatchTimer()` 함수
   - 설정 모달에 "시청 통계" 섹션: 총 시청 시간 + 채널별 시청 시간 Top 10
   - `formatWatchTime()`, `buildWatchStatsHtml()` 함수
   - Firebase 동기화에 `watchLog` 포함

6. **채널 영상 알고리즘 개선**
   - **기존**: RSS 피드 (최근 15개, 조회수/시간 없음)
   - **변경**: 채널 페이지 스크래핑 (`/channel/{id}/videos?view=0&sort=p`) → `ytInitialData` 파싱
   - `fetchChannelPopularVideos()`: richGridRenderer → videoRenderer 추출
   - Shorts 필터링: 60초 미만 + `thumbnailOverlayTimeStatusRenderer.style === 'SHORTS'` + URL `/shorts/` 체크
   - 조회수 (`viewCountText`), 영상 길이 (`lengthText`) 메타정보 카드에 표시
   - 영상 길이 배지 (썸네일 우하단)
   - Fallback: 스크래핑 실패 시 기존 RSS 대체, 라벨로 구분 (인기 영상 / 최신 영상 RSS)
   - `fetchChannelRSS()`는 자동 분류용으로 유지

### 버그 수정

1. **채널 URL 추가 시 아이콘 깨짐**
   - 증상: URL로 채널 추가 시 썸네일이 빈 값 (`''`)
   - 원인: `addChannelFromUrl()`에서 채널 페이지 아바타 추출 로직 없음
   - 수정: `resolveChannelInfo()` 함수 추가 — 채널 페이지 HTML에서 `"avatar":{"thumbnails":[...]}` 패턴으로 추출
   - 추가: 빈 썸네일 채널 카드에 📡 플레이스홀더 표시

2. **CORS 프록시 안정성 (모바일 타임아웃)**
   - 증상: 모바일에서 채널 검색/추가 시 간헐적 실패
   - 수정: CORS 프록시 3개로 확대 (`corsproxy.io`, `codetabs.com`, `allorigins.win`), AbortController 15초 타임아웃, 빈 응답 체크

3. **모바일 채널 검색 완전 실패** ⭐
   - 증상: 모바일에서 "고양이", "교양이" 등 검색 시 항상 "검색에 실패했습니다" 표시
   - **근본 원인**: CORS 프록시가 모바일 User-Agent를 YouTube에 그대로 전달 → YouTube가 **모바일용 페이지** 반환
     - 모바일 페이지의 JSON이 `\x7b\x22...` hex escape 인코딩 (기존 `{...}` 정규식 매칭 실패)
     - 채널 데이터가 `channelRenderer` 대신 `compactChannelRenderer` 사용 (필드 구조도 다름)
   - 수정:
     - `decodeJsEscapes()`: `\x`, `\\`, `\/`, `\u` 등 모든 JS escape 시퀀스 디코딩
     - `extractYtInitialData()`: 데스크탑 (`{...}`) + 모바일 (`'\x7b...'`) 두 형식 모두 처리
     - `parseChannelResults()`: `channelRenderer` + `compactChannelRenderer` 양쪽 지원
     - `fetchChannelPopularVideos()`도 공통 함수 사용

4. **PWA 스플래시 아이콘 미변경**
   - 증상: 앱 시작 시 중간에 보이는 아이콘이 이전 것
   - 수정: `manifest.json`에 `purpose: "maskable"` 아이콘 추가, `apple-touch-startup-image` 메타태그 추가
